学习C语言过程中发现的一些问题

Created2022-10-10|Updated2022-10-10|编程语言
|Word count:586|Reading time:2min|Post View:|Comments:

发现的问题

  • 以下面这个题目为例，在将转换后的小写英文字母转换为十进制的ASCII码时

一开始，我尝试编写的代码是这样的,编译之后，出现了这样的错误。

   char ch,AI;
   int b;
   ch=getchar();
   AI = 32 + ch;
   b = 32 + 'ch';
printf("%c,%d",AI,b)

发现，并没有输出正确的ASCII码值。于是我认为可能是编译器将**’ch’中的ch当成了ch组成的字符串，并没有将它看成我所定义的ch变量**。(但是为什么结果为25480，目前我还无法给出正解)发现错误之后，我将代码更改为以下内容

char ch,AI;
int b;
ch=getchar();
AI = 32 + ch;
b = (int)(32 + ch);

通过强制将32 + ch转换为整型，再赋值给变量b，编译之后，成功实现目标。

  • 还是上面题目为例，关于if语句我也有了新的认识。

       if(97<=b<=122)
   {
	printf("%c,%d",AI,b);	
   }
else
{
	printf("Input Data error!");
}

    一开始，我认为这样在if()中直接写不等式组机器是能够识别出来的，但我还是太天真了，机器不能识别出这是一串什么东西，编译后的结果总是不对，于是我又改成了这样

       if(b<=122,b>=97)
   {
	printf("%c,%d",AI,b);	
   }
else
{
	printf("Input Data error!");
}

    不出意外，还是错误的，无奈之下我只能翻阅教材，之后才发现原来要在中间加上**&&**才可以被机器识别，于是我将代码改成了这样

       if(b<=122 && b>=97) 
   {
	printf("%c,%d",AI,b);	
   }
else
{
	printf("Input Data error!");
}

    成功运行。

以上是我在完成作业时遇到的一些问题（其实还有很多小问题）

对于上面这题，还有更简单方法

#include<stdio.h>
#include<math.h>
int main()
{
    char ch,AI;
    ch=getchar();
    AI = 32 + ch;
    if(AI<=122 && AI>=97)	//判断字符类型
    {
		printf("%c,%d",AI,AI);	//在这里直接将AI变量以不同的类型输出
    }
	else
	{
		printf("Input Data error!");
	}
    return 0;
}

反思

通过这些问题，我也意识到，在编写代码时一定要用紧密的逻辑，仔细研究所给题目，设计算法，这样才能更好的提升自己的代码能力。
